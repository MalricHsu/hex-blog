## 4. Nuxt 核心功能

### 4-1 檔案路由（File-based Routing）

`pages/` 底下的檔案會**自動變成路由**，不用手寫 router：

| 檔案 | 網址 |
| --- | --- |
| `pages/index.vue` | `/` |
| `pages/service.vue` | `/service` |
| `pages/blog/index.vue` | `/blog` |
| `pages/blog/[id].vue` | `/blog/1`、`/blog/2`（動態路由） |

- `index.vue` 代表「該資料夾本身」。
- `[id].vue` 的方括號是**動態參數**，用 `useRoute().params.id` 取得。

### 4-2 Layouts（版型）

把共用框架（Nav / Footer）抽到 layout，頁面只放內容：

```vue
<!-- layouts/default.vue -->
<template>
  <Nav />
  <slot />     <!-- 頁面內容塞這裡 -->
  <Footer />
</template>
```

```vue
<!-- app.vue -->
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
  <ToastContainer />
</template>
```

某頁要用不同版型（例如 contact 不要 footer）：

```js
definePageMeta({ layout: "no-footer" });
```

### 4-3 自動匯入（Auto-imports）

Nuxt 會自動匯入，**不用寫 import**：

- `components/` 的元件
- `composables/`、Vue API（`ref`、`computed`、`onMounted`…）
- `stores/` 的 Pinia store（複數資料夾名才會自動匯入）
- `server/utils/`（僅伺服器端）
- `shared/utils/`（前後端都會）

### 4-4 資料取得（useFetch / useAsyncData）

```js
const { data: service } = await useFetch("/api/service");
```

- **SSR 階段就在伺服器抓好資料**，吐含資料的 HTML（利於 SEO / 首屏）。
- 回傳的是 ref，資料回來自動更新畫面。
- **取代了** `onMounted + fetch + ref` 那一套，所以本專案抓 API 都不用 `onMounted`。
- `useAsyncData(key, fn)` 用在「自訂抓取邏輯」，例如 `queryCollection`。

> `onMounted` 仍會用在「一定要在瀏覽器、DOM 準備好才能做的事」，例如初始化香草版第三方套件。

### 4-5 標題與 SEO（useHead / useSeoMeta）

```js
// 各頁
useHead({ title: "服務項目" });
```

```ts
// nuxt.config.ts — 全域樣板
app: {
  head: {
    titleTemplate: "%s | Nelson Blog",
    title: "Nelson Blog",
    link: [{ rel: "icon", type: "image/svg+xml", href: "/favicon.svg" }],
  },
},
```

→ 各頁 `useHead({ title })` 會代入 `%s`，變成「服務項目 | Nelson Blog」。

### 4-6 伺服器 API（Server Routes）

`server/api/` 底下的檔案就是 API 端點：

```js
// server/api/send-footerContact.post.js
export default defineEventHandler(async (event) => {
  const body = await readBody(event);          // 讀請求 body
  const result = contactSchema.safeParse(body); // 伺服器端再驗一次
  if (!result.success) {
    throw createError({ statusCode: 400, message: "..." });
  }
  return { success: true, message: "聯繫表單已送出" };
});
```

- 檔名 `.post.js` 代表只接受 POST。
- `readBody`、`createError` 是 Nitro（Nuxt 伺服器引擎）自動匯入的工具。

### 4-7 shared 目錄（前後端共用）

Nuxt 4 的 `shared/utils/` 會**同時自動匯入到前端元件與伺服器**，最適合放「前後端都要用」的東西，例如驗證 schema。本專案把 `contactSchema` 放這裡，確保前端先驗、後端再驗都用同一份規則。

### 4-8 public（靜態檔）

`public/` 的檔案會放在網站根目錄。例如 `public/favicon.svg` → `/favicon.svg`。本專案自製了 N 字母 SVG favicon。

## 5. Nuxt Content 部落格系統

### 5-1 定義 collection

```ts
// content.config.ts（專案根目錄）
import { defineContentConfig, defineCollection, z } from "@nuxt/content";

export default defineContentConfig({
  collections: {
    blog: defineCollection({
      type: "page",          // page = 會對應到網址
      source: "blog/**/*.md", // 抓 content/blog 底下
      schema: z.object({      // frontmatter 自訂欄位
        image: z.string(),
        categories: z.array(z.string()),
        date: z.string(),
        dateFormatted: z.string(),
        views: z.number(),
        shares: z.number(),
      }),
    }),
  },
});
```

### 5-2 撰寫文章

`content/blog/1.md`（檔名即網址 → `/blog/1`）：

```md
---
title: 文章標題
image: 封面圖網址
categories: [UIUX 設計, 前端技術]
date: "2025-10-16"
dateFormatted: "Oct 16, 2025"
views: 110
shares: 2
---

## 內文標題

支援標準 Markdown、表格、圖片，也能內嵌 HTML 搭配 Bootstrap class。
```

### 5-3 列表頁與詳情頁

```js
// 列表：撈全部
const { data: blogs } = await useAsyncData("blog-list", () =>
  queryCollection("blog").all()
);
```

```js
// 詳情：用網址路徑撈單篇
const route = useRoute();
const { data: blog } = await useAsyncData(route.path, () =>
  queryCollection("blog").path(route.path).first()
);
```

```vue
<!-- 渲染 Markdown -->
<ContentRenderer v-if="blog" :value="blog" />
```

### 5-4 重點

- `queryCollection` 在「元件」是自動匯入；伺服器 route 要用 `queryCollection(event, "blog")`（簽名不同）。
- `<ContentRenderer>` 把 md 轉成 HTML，不用自己 `v-html`。
- 渲染出的 HTML 樣式要靠 **`:deep()`** 才套得到（見樣式章節）。

## 6. Vue 3 觀念整理

### 6-1 `<script setup>` 與響應式

```js
const email = ref("");          // 基本型別用 ref，存取用 .value
const form = reactive({ ... });  // 物件用 reactive
const filtered = computed(() => ...); // 衍生資料用 computed，依賴變動自動重算
```

> 模板裡 ref 會**自動解包**（不用 `.value`）；`<script>` 裡要 `.value`。

### 6-2 Props（父傳子）

```js
const props = defineProps({
  type: { type: String, default: "success" },
  msg: { type: String, default: "" },
});
```

子元件要用的 prop **一定要在 `defineProps` 宣告**，否則拿到 `undefined`。

### 6-3 `:` 的有無（v-bind）

| 寫法 | 解讀 |
| --- | --- |
| `type="success"` | 純字串字面值 |
| `:msg="successMsg"` | 當 JS 執行 → 讀變數的值 |

口訣：**要「讀變數」就加 `:`，要「固定字串」就不加。**

### 6-4 事件

```vue
<form @submit.prevent="submitForm">   <!-- .prevent 阻止表單預設重整 -->
<button @click="isOpen = !isOpen">
```

### 6-5 v-for / v-if / :key / :class

```vue
<li
  v-for="(item, index) in list"
  :key="item.id"
  :class="{ active: item.id === selectedId }"
>
```

- `:class="{ 類別: 條件 }"` → 條件成立才加該 class。
- `v-for` 與 `:key` 放在**重複的那個元素**上。

### 6-6 Slot（插槽）

`<slot />` 是「把外層包進來的內容放在這個位置」的佔位符，layout 的頁面內容就是塞進 `<slot />`。

### 6-7 Transition / TransitionGroup

Toast 進出動畫用 `<TransitionGroup>`：

```vue
<TransitionGroup tag="div" name="toast">
  <FormAlert v-for="t in toast.toasts" :key="t.id" ... />
</TransitionGroup>
```

```css
.toast-enter-from, .toast-leave-to { opacity: 0; transform: translateX(100%); }
.toast-enter-active, .toast-leave-active { transition: all 0.3s ease; }
```

## 7. Pinia 狀態管理（全域 Toast）

### 流程：store → container → 任何元件呼叫

```
表單呼叫 toast.success("...") → store 的 toasts 陣列被 push
   → ToastContainer 的 v-for 監聽 store，自動多畫一個通知
   → ToastContainer 放在 app.vue，全站都看得到
   → 3 秒後 store 移除該筆 → 通知自動消失
```

### Store（setup 寫法）

```js
// app/stores/toast.js
export const useToastStore = defineStore("toast", () => {
  const toasts = ref([]);                    // state 用 ref
  const addToast = (type, msg) => {
    const id = Date.now();
    toasts.value.push({ id, type, msg });
    setTimeout(() => removeToast(id), 3000);
  };
  const success = (msg) => addToast("success", msg);
  const error = (msg) => addToast("error", msg);
  const removeToast = (id) =>
    (toasts.value = toasts.value.filter((t) => t.id !== id));
  return { toasts, success, error, removeToast }; // return 出去才用得到
});
```

### 使用

```js
const toast = useToastStore();
toast.success("訂閱成功！");   // ⚠️ action 是「呼叫」，不是 toast.success = "..."
```

### 重點

- store 放 `app/stores/`（**複數**）才會自動匹入。
- state 用 `ref`；action 是函式，**用括號呼叫**。
- 安裝：`npm i pinia @pinia/nuxt` + 在 `nuxt.config` 的 `modules` 加 `@pinia/nuxt`。

## 8. 表單驗證流程（前後端共用 Zod）

### Schema（放 shared，前後端共用）

```js
// shared/utils/schema.js
import { z } from "zod";
export const contactSchema = z.object({
  name: z.string().min(2, "請輸入姓名"),
  phone: z.string().regex(/^09\d{8}$/, "手機號碼格式不正確"),
  email: z.string().email("請輸入正確格式"),
  note: z.string().optional(),
});
```

### 前端送出流程

```js
const submitForm = async () => {
  // 1. 前端先驗
  const result = contactSchema.safeParse({ name, phone, email, note });
  if (!result.success) {
    toast.error(result.error.issues.map((i) => i.message).join("、"));
    return;
  }
  // 2. 通過才送出
  sending.value = true;
  try {
    const res = await axios.post("/api/send-footerContact", result.data);
    toast.success(res.data.message);   // axios 資料在 res.data
  } catch (error) {
    toast.error(error.response?.data?.message || "送出失敗");  // axios 錯誤在 error.response.data
  } finally {
    sending.value = false;
  }
};
```

### 重點

- **前後端都驗一次**：前端給即時回饋，後端防繞過。共用 schema 確保規則一致。
- `safeParse` 不會丟錯，回傳 `{ success, data/error }`。
- axios：成功資料在 **`res.data`**，錯誤在 **`error.response.data`**。

## 9. 樣式系統（Bootstrap 客製 SCSS）

### 9-1 客製主題

`_variables.scss` 自訂了 `$theme-colors`（primary-50/500/900、neutral-0/500/700/900…），Bootstrap 會自動生成 `bg-primary-900`、`text-neutral-0`、`btn-neutral-700` 等 utility 與按鈕 hover。

`_base.scss` 自訂了字級系統 `.fs-heading-*`、`.fs-paragraph-*`。

### 9-2 scoped 樣式與 `:deep()`

元件 `<style scoped>` 只作用在自己。但要套到**子元件根元素**或 **`<ContentRenderer>` 動態產生的內容**（如文章表格），必須用 `:deep()` 穿透：

```css
:deep(table) { width: 100%; }
:deep(thead) { background-color: #24140b; }
```

### 9-3 全域重設的副作用

`_base.scss` 有 `h1~h6, p { margin: 0 }`，導致 Markdown 內文間距被歸零，所以詳情頁要用 `:deep(p) { margin-top: 16px }` 把間距補回來。

### 9-4 在 Markdown 裡用樣式

MDC 的 `{.class}` 在本專案沒被解析，改用**原生 HTML + Bootstrap class**（Content 會原樣渲染、全域 class 會生效）：

```html
<p class="mb-12">需要更多下方間距的段落</p>
<div class="bg-primary-50 text-center p-8 rounded">引言框</div>
```

## 10. RWD 響應式策略

### 10-1 「補 base class」模式

Bootstrap 的 `xxx-md-OO` 只在該斷點以上生效。看到只有 `-md-` 的間距/排版，就**補一個沒斷點的 base 值**當手機預設：

```
py-md-20  →  py-10 py-md-20       （手機 py-10，桌機 py-20）
mb-md-6   →  mb-4 mb-md-6
d-flex gap-6（卡片列）→ 加 flex-wrap 或 flex-column
```

### 10-2 斷點整體上移（md → lg）

平板（768~991）在 `md` 斷點會擠到跑版，因此把全站結構斷點**從 md（768）上移到 lg（992）**，讓平板也用手機堆疊版型：

- `col-md-*` → `col-lg-*`
- `flex-md-row` → `flex-lg-row`
- `navbar-expand-md` → `navbar-expand-lg`
- scoped media query `(max-width: 768px)` → `991.98px`、`(min-width: 768px)` → `992px`

> 用 `991.98px` 而非 992，是 Bootstrap 慣例，避免和 `min-width: 992px` 在整數點重疊。

### 10-3 響應式字級

大英文標題（如 `SUBSCRIPTION` 80px）在窄螢幕會爆版，於 `_base.scss` 用 media query 分階縮小：

```css
@media (max-width: 1200px)   { .fs-heading-xxx-large { font-size: 56px; } }
@media (max-width: 991.98px) { .fs-heading-xxx-large { font-size: 40px; } }
```

### 10-4 橫向滑動與隱藏捲軸

手機的分類列、卡片列改成可橫滑，並隱藏捲軸：

```css
.scroll { overflow-x: auto; scrollbar-width: none; }
.scroll::-webkit-scrollbar { display: none; }
```

### 10-5 換頁版面跳動

`/contact`（短頁、no-footer）沒捲軸、其他頁有，換頁時捲軸消失造成左右跳動。解法：

```css
html { scrollbar-gutter: stable; }  /* 永遠預留捲軸空間 */
```

## 11. 元件抽離與複用

把重複的 UI 抽成元件，「改一處、全站同步」：

- **FormAlert**：成功/失敗的 alert 外觀，傳 `type` + `msg`。
- **ToastContainer**：讀 store、用 v-for 渲染多個 FormAlert。
- 其他可抽：PageBanner、SocialLinks 等。

**分工概念**：store 管資料、FormAlert 管畫面、ToastContainer 把兩者接起來。

## 12. 互動細節

- **hover**：卡片浮起（`translateY` + `box-shadow`）、社群圈填色、箭頭鈕放大、nav 連結變色；都加 `transition` 讓變化平滑。
- **FAQ 手風琴**：不靠 Bootstrap JS，用 Vue 的 `openIndex` 狀態 + `max-height` 過渡控制展開。
- **漢堡選單**：用 `isOpen` 切換 Bootstrap 的 `.show` class（顯示交給 Bootstrap CSS），點連結後 `@click="isOpen = false"` 自動收合。
- **麵包屑分隔線**：Bootstrap 用 `::before` 畫，scoped 要 `:deep()` 改色。

## 13. 重點觀念與踩雷紀錄

| 雷 | 正解 |
| --- | --- |
| 箭頭函式有 `{}` 卻沒 `return` | 大括號主體必須 `return`，或拿掉括號自動回傳 |
| `v-if="(item) in list"` | 迴圈是 `v-for`，不是 `v-if` |
| `useFetch` 是 ref，`.value` 漏了 | `<script>` 裡要 `projects.value` |
| `toast.success = "..."` | action 要呼叫：`toast.success("...")` |
| `:msg="{ successMsg }"` | 多包大括號變物件，應 `:msg="successMsg"` |
| `axios.post` 漏 `await` | 沒 await，`res` 是 Promise，`res.data` 為 undefined |
| `res.message` / `error.data` | axios 是 `res.data.message` / `error.response.data.message` |
| `server/utils` 在元件拿不到 | 前後端共用要放 `shared/utils` |
| Prettier 消不掉紅點 | 紅點是 ESLint/Volar，Prettier 只管排版 |
| `defineProps` 漏宣告 prop | 沒宣告的 prop 在元件內是 undefined |
| inline `:style` 無法寫 media query | 改用 scoped class + `@media` |

## 14. 何時需要重啟 dev server

熱更新（HMR）對「元件 template / script / style」即時生效，但以下**結構性變動要重啟**：

- 修改 `nuxt.config.ts`（加 module、改 head…）
- 安裝新套件 / 加 Pinia、Content 等 module
- 新增 `shared/`、`stores/`、`layouts/` 等需要重建自動匯入索引的資料夾
- 新增 `content.config.ts` 或 Content collection 變更
